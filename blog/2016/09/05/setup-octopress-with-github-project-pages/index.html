Setup Octopress With Github Project Pages

Sep 5th, 2016 3:17 am

Since I already have my GitHub Page, I want to setup Octopress blog with URL: absoluteyl.github.io/blog. It turns out there’s a function called “GitHub Project Pages”, all you need to do is to start create a branch named “gh-pages”, GH Pages will generate a subdirectory for the repo automatically. But Octopress Deploy document somehow confuse me on how to do it, and I took a lot more time than I thought it should be. So I think I should wrote down the entire process for this.

Setup Octopress

Just download octopress from its GitHub Repo

And then, install bundler and related gems

Install default Octopress theme

Note: There’re a lot out-of-date tutorials said that Octopress only supports Ruby 1.9.2, and tells you to use this version instead. According to my experience, Octopress actually runs with later Ruby version now.

Create Project Repo

In your GitHub account, create a new repo with name you want to use as subdirectory, then copy repo url to clipboard. While create the new repo, make sure not to tick option “Initialize this repository with a README”, we need an empty repo.

Configure your Octopress

The main config file of Octopress is in _config.yml. For my blog, I only changed title, subtitle, author in the begining. You can refer to Octopress Document for detail settings

Deploy with GitHub Project Pages

Now we’re ready to deploy

This command will let you insert your GitHub url (the one we just copied), rename imathis/octopress from “origin” to “octopress” (so you won’t push some yourown stuff accidently) and then setup branch “gh-pages” as default branch in your repo.

Following commands will generate your blog files into _deploy folder, add them to git, commit and push them up to your GitHub repo. At last, you need to setup new origin as your GitHub repo and tell git to always push local master to remote gh-pages.
